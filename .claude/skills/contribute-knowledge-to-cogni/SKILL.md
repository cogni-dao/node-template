# Contribute Knowledge to Cogni

Read `docs/guides/contribute-knowledge.md`.

Default to silence. If the finding is durable, reusable, and not obvious from code or existing docs, refine an existing entry first. Write a new atomic entry only when no existing entry fits.

Never include secret values, private user data, raw credentials, or transient PR status.
